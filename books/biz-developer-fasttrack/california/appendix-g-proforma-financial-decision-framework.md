| --- | --- |

| Holding cost ceiling | Monthly hold ≤1% of expected gross profit |
| Time-value test | 6-month delay must not drop IRR below 15–20% hurdle |

## Level 1: The 5-Minute Napkin Test

This is the first financial gate. You should be able to run this test using publicly available data (Zillow, Redfin, county assessor) within 5 minutes. If the deal does not clear this gate, do not proceed to Level 2 or the Site Constraint Analysis.

### 1A. Residential Napkin Test (SFR / SFR+ADU / Multifamily)

| Line Item | Amount |
| --- | --- |
| **REVENUE** | |
| A. Estimated ARV (or stabilized value for MF) | |
|     Source: _______ comps at $_______/SF avg | |
| **COSTS** | |
| B. Acquisition (purchase price + closing) | |
| C. Hard Costs (construction $/SF × total SF) | |
|     Construction $/SF used: $_______ | |
|     Total SF: _______ | |
| D. Soft Costs (design, engineering, permits = 15–20% of C) | |
| E. Government Fees (impact + connection + permit) | |
| F. Holding Costs (monthly hold × months) | |
|     Monthly: $_______  ×  Months: _______ | |
| G. Contingency (10% of C+D+E) | |
| H. Selling Costs (5–6% of A: commissions + transfer tax) | |
| **TOTAL ALL-IN COST (B+C+D+E+F+G+H)** | |
| **GROSS PROFIT (A − Total)** | |
| **GROSS MARGIN % (Profit ÷ Total)** | |

> **DECISION RULE**: SFR Spec: Gross margin must be ≥15% to proceed. SFR+ADU: ≥20%. Multifamily: ≥20%. If margin is 10–15%, project is marginal—only proceed if you have a clear path to reduce costs or increase value.

### 1B. Hotel Napkin Test

| Line Item | Amount |
| --- | --- |
| **REVENUE (Stabilized Annual)** | |
| A. Number of Keys | |
| B. Average Daily Rate (ADR) | |

| G. NOI (F × 35–40% margin for select-service) | |
| **VALUATION** | |
| H. Cap Rate (typically 7–9% select-service) | |
| I. Stabilized Value (G ÷ H) | |
| **COST** | |
| J. Total Development Cost ($/key × keys) | |
|     Cost per key used: $_______ | |
| **DEVELOPMENT SPREAD (I − J)** | |
| **YIELD ON COST (G ÷ J)** | |

> **DECISION RULE**: Hotel: Yield on cost must exceed cap rate by ≥1.5% (e.g., 7% cap = need 8.5%+ yield on cost). Development spread must be ≥25% of total cost. If yield on cost is below cap rate, you are destroying value.

### 1C. Napkin Test Decision

```
☐ PASS — Proceed to Level 2

☐ FAIL — Kill or Restructure
  Below threshold. Either renegotiate acquisition or identify cost reduction.
```

## Level 2: Development Pro Forma

This is the detailed financial model. Complete this in parallel with the Site Constraint Analysis (Appendix E, Sections 1–4). The constraint analysis tells you **what you can build**; this tells you **what it costs and what it is worth**. Every line item requires a source or basis—no guessing.

> **CASE STUDY LESSON**: On the Peninsula project, the original pro forma assumed a 12–15 month permit + build timeline. Actual timeline exceeded 36 months. The holding cost alone consumed more than the projected profit margin. Every pro forma must be stress-tested against a realistic worst-case timeline.

#### Break-Even & Death Point Analysis

> ⚠ **The Death Point** is the month at which cumulative holding costs consume your entire projected profit. If your worst-case timeline extends past the death point, the deal has unacceptable risk without restructured financing.

| Break-Even Metric | Value |
| --- | --- |
| Projected Gross Profit (base case) | |
| Monthly Burn Rate | |
| Death Point Month (Profit ÷ Monthly Burn) | |
| Loan Maturity Month (from acquisition) | |
| Loan Maturity + All Extensions | |
| Worst-Case Total Duration | |

> **DECISION RULE**: If Death Point Month < Worst-Case Duration: **RESTRUCTURE FINANCING** before proceeding.

> **DECISION RULE**: If Loan Maturity + Extensions < Worst-Case Permit Timeline: **RESTRUCTURE FINANCING** before proceeding.

> **DECISION RULE — Ideal**: Death Point should be at least **2× your worst-case total duration**. This gives you a margin of safety for the unexpected.

## Level 3: Stress Test & Kill Criteria

This is the final financial gate. Take your base-case pro forma and systematically degrade each assumption. If the project survives the stress test, it is a GO. If any single stress case kills the deal, you must either mitigate that specific risk or walk away.

### 3A. Single-Variable Stress Tests

For each variable below, hold all other assumptions constant and stress only the variable shown. Record the result and determine if the project still meets minimum thresholds.

### 3B. Combined Stress Test (Worst Realistic Case)

This is the scenario that actually happens in a downturn: multiple things go wrong simultaneously. Apply **ALL** of the following at once: ARV -10%, hard costs +20%, timeline +50%.

| Combined Stress Metric | Value |
| --- | --- |
| Stressed Revenue (ARV at -10%) | |
| Stressed Cost (+20% hard costs) | |
| Stressed Holding (+50% timeline) | |
| Stressed Gross Profit | |
| Stressed Gross Margin % | |

> **DECISION RULE — Combined Stress Test**: If gross margin is still positive, the project has a reasonable margin of safety. If margin goes negative under combined stress, the project is a leveraged bet on everything going right. Proceed only if you can absorb the downside.
