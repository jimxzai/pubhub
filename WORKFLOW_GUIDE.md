# PubHub 工作流指南

> 使用 Claude Skills 和 Coworkers 的完整工作流程

## 概述

本项目使用5个 Claude Skills (AI协作者) 构建完整的出版流水线：

```
阅读原文 → 写初稿 → 注疏 → AI映射 → 评估 → 润色 → 发布
              ↓        ↓        ↓        ↓       ↓
            你写    注疏师  AI战略家  总编辑  校对神
```

---

## 第一部分：每日工作流

### Step 1: 创建每日笔记

```bash
npm run new-note
```

这会创建 `daily-notes/drafts/YYYY-MM-DD.md`，使用标准模板。

### Step 2: 阅读与写作 (60分钟)

打开笔记文件，填写四个必需部分：

```markdown
## 今日原文摘要
> 兵者，国之大事，死生之地，存亡之道，不可不察也。
> (Warfare is the greatest affair of state...)

## 我的理解与心得
[你的300-500字思考]

## 2025年AI时代的对应
[具体的AI案例映射]

## 对7年大业的启发
[可应用的场景]
```

### Step 3: 使用注疏师获取深度注疏 (可选)

在 Claude Code 中运行：

```
/annotate
```

然后告诉注疏师你需要什么：

**示例对话**:
```
你: 请为《孙子兵法》始计篇第一段提供注疏

注疏师:
## 原文（中英对照）
### 中文
> 孙子曰：兵者，国之大事...

### 英文
> Sun Tzu said: Warfare is the greatest affair of state...

## 历代注疏
### 1. 曹操注
> 国之安危在此，不可不察也。
**现代理解**: ...

### 2. 杜牧注
> ...

## 关键词解析
**1. 兵**
- 原文: 军队、战争
- 现代: 可延伸为竞争、博弈
- 演变: 从具体战争到抽象策略
```

### Step 4: 使用AI战略家映射现代案例 (可选)

```
/ai-parallels
```

**示例对话**:
```
你: 请将"兵者，国之大事"映射到AI时代

AI战略家:
## 核心战略原则
### 原则1: 战略决策的不可逆性
> AI研发如同古代开战，一旦公开发布便不可收回...

## AI时代战例映射
### 原则1 → 2024年战例
**案例**: OpenAI GPT-4发布策略
**时间**: 2023年3月
**映射分析**:
- 相似之处: 慎重决策，多次内部测试
- 差异之处: 数字产品可迭代更新
- 成败关键: 首发优势 vs 安全对齐

## 战略推演 (2025-2035)
**短期**: AGI竞争加剧...
**中期**: 监管框架成型...
**长期**: 人机协作常态化...
```

### Step 5: 使用总编辑分析评估

```
/master-editor
```

粘贴你的笔记，获取客观评估：

**输出示例**:
```markdown
## 内容分类
- 书名: 孙子兵法
- 章节: 始计篇
- 核心主题: 战略决策、风险评估

## 质量评估
- 原文摘要: 4/5
- 个人理解: 3/5 (可加深)
- AI对应: 4/5
- 实践启发: 3/5
- **总分**: 14/20

## 改进建议
1. 加入更多个人独特洞见
2. AI案例可补充具体数据
3. 实践启发部分过于笼统

## 建议标签
`#孙子兵法` `#始计篇` `#战略决策` `#AI竞争` `#风险管理`
```

### Step 6: 使用校对神润色

```
/proofread
```

**输出示例**:
```markdown
## 润色建议
### 整体评价
- 流畅度: 4/5
- 准确性: 4/5
- 专业性: 4/5

### 具体修改
**第2段**
- 原文: "这个很重要"
- 建议: "此点至关重要"
- 理由: 避免口语化表达

## 引文核对
### ✅ 准确的引用
- "兵者，国之大事" — 《孙子兵法·始计篇》

### ⚠️ 需要核对的引用
- OpenAI GPT-4发布日期 — 请确认为2023年3月14日
```

### Step 7: 发布

```bash
# 移动到已发布目录
mv daily-notes/drafts/2026-01-18.md daily-notes/published/

# 提交
git add .
git commit -m "📝 Daily note: 2026-01-18 孙子兵法始计篇"
git push
```

## 第二部分：周度工作流

### 每周六：生成周总结

```
/publish --type weekly
```

**输入**: 本周 `daily-notes/published/` 中的7篇笔记

**输出** (1000-2000字):
```markdown
# 2026年第3周总结

## 本周核心主题
1. **战略决策的慎重性** — 从孙子"兵者国之大事"看AI发布策略
2. **知己知彼的现代诠释** — 竞争情报与AI模型评测
3. **奇正相生的创新思维** — 开源与闭源的动态博弈

## 本周金句精选
> "知彼知己，百战不殆" — 《孙子兵法·谋攻篇》
> "Be quick, but don't hurry." — John Wooden

## 跨书关联发现
- 孙子的"上兵伐谋"与《资治通鉴》中诸葛亮的外交策略
- 圣经箴言的"谋士众多，所谋乃成"与集体智慧

## 下周阅读计划
- 孙子兵法：作战篇、谋攻篇
- 圣经：约翰福音第3章
```

## 最佳实践

### 1. 保持节奏
- 每日固定时间阅读写作 (如早上7:00-8:30)
- 周六固定生成周报
- 月末固定生成月报

### 2. 质量优先
- 使用 `/master-editor` 确保每篇笔记达到14/20分以上
- 使用 `/proofread` 确保引文准确

### 3. 渐进积累
- 不追求完美，追求持续
- 每篇笔记有1个独特洞见即可
- 7年2557篇，日积月累

### 4. 协作者组合
- **日常**: 注疏师 + 总编辑
- **深度研究**: 注疏师 + AI战略家 + 总编辑 + 校对神
- **快速发布**: 总编辑 + 校对神
