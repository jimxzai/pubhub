## 🔧 Git 命令参考

### 日常提交流程
```bash
# 1. 查看状态
git status

# 2. 添加文件
git add .

# 3. 提交（使用规范的消息格式）
git commit -m "📝 Daily note: YYYY-MM-DD

简短描述...

🤖 Generated with Claude Code
Co-Authored-By: Claude <noreply@anthropic.com>"

# 4. 推送到 GitHub
git push
```

### 检查同步状态
```bash
# 查看远程状态
git fetch origin
git status

# 查看提交历史
git log --oneline -10

# 查看文件变更
git diff
```

## 📞 故障排除

### 如果推送失败
```bash
# 1. 检查远程连接
git remote -v

# 2. 测试 SSH 连接
ssh -T git@github.com

# 3. 拉取远程更改
git pull origin main

# 4. 重新推送
git push
```

### 如果有冲突
```bash
# 1. 查看冲突文件
git status

# 2. 手动解决冲突
# 编辑冲突文件，删除冲突标记

# 3. 标记为已解决
git add .
git commit -m "解决冲突"
git push
```
